now that we have completed analysis of our first argument, that the time of day affects bike rentals, we can begin looking into the data of how weather patterns affect bike rentals. in order to make sure that we are looking only at how weather affects the data and not time, we will only look at data from the hour 17:00 as this has the highest number of average rentals.

in order to effectively keep time as a constant we are going to create a new data set that has only the data of 1700 hours.

bike_1700 <- subset(bike_data_clean, hr == 17)

we can now see how many observations we have in this data set

nrow(bike_1700)
## [1] 730

we have 730 observations. we can now use multiple linear regression to see how weather situation, apparent temperature, humidity, and wind speed factor into the overall count of rented bikes. in this case we are taking weather situation to mean how the overall weather of the time is with 1 = clear/partly cloudy 2 = mist + cloudy 3 = light snow/ light rain 4 = heavy rain/ ice/ snow It should also be noted that we are choosing apparent temperature over actual temperature as apparent temperature will have a higher impact on human behavior of bike rental

however to test our model we can run a prediction and see how it matches up to the real data. lets take a day that is a normal working day (1), in clear weather (is_rainy = 0), with average temp and average humidity. first we will have to find the average temp and average humidity and then we can fit them into our model.

mean_atemp <- mean(bike_data_clean$atemp)
mean_hum <- mean(bike_data_clean$hum)

print(paste("average normalized temp:", round(mean_atemp, 3)))
## [1] "average normalized temp: 0.476"
print(paste("average normalized humidity:", round(mean_hum, 3)))
## [1] "average normalized humidity: 0.627"

now that we have the averages we can make our predictions for an average non rainy working day at 1700 hours

new_day <- data.frame(
  hr = 17,
  workingday = 1,
  atemp = mean_atemp,
  hum = mean_hum,
  is_rainy_master = 0
)

predicted_count <- predict(master_model, newdata = new_day)
print(paste("predicted rentals at 5 pm:", round(predicted_count)))
## [1] "predicted rentals at 5 pm: 446"

the model shows that the predicted number of bikes for an average day at 5 pm in clear weather is 446, lets see how this compares to our actual data. to do this we will find a day that matches our prediction model with a +/- .05 difference in each catagory

real_matches <- subset(bike_data_clean,
                       hr == 17 &
                         workingday == 1 &
                         is_rainy_master == 0 &
                         atemp >= (mean_atemp - 0.05) & atemp <= (mean_atemp + 0.05) &
                         hum >= (mean_hum - 0.05) & hum <= (mean_hum + 0.05))
nrow(real_matches)
## [1] 10
#this will show how many days actually match the criteria and now we can average them
mean(real_matches$cnt)
## [1] 470.7

we can see that we have 10 matches in our entire data set for these conditions and these 10 days all have an average of 470.7 rentals. we can now find the percent error of our model to our actual data to see how close we are in the accuracy of our model

actual_avg <- mean(real_matches$cnt)
predicted_val <- 446
percent_error <- (abs(actual_avg - predicted_val) / actual_avg)*100

print(paste("actual average rentals:", round(actual_avg, 2)))
## [1] "actual average rentals: 470.7"
print(paste("predicted rentals:", predicted_val))
## [1] "predicted rentals: 446"
print(paste("percent error:", round(percent_error, 2), "%"))
## [1] "percent error: 5.25 %"

we got a percent error of 5.25% which means our model is extremely strong as a predictor ;)
